# 🚀 [서비스사용량통합대시보드] 마스터 대시보드

> **💡 업데이트 규칙:**
> 기획이나 스택이 변경될 경우 기존 내용을 지우지 말고, 취소선(`~~내용~~`)과 날짜를 기입하여 변경 이력을 남깁니다.

## 1. 프로젝트 개요 (Overview)
* **목적 및 배경:** 실제 런칭한 서비스들이 사용하고 있는 백엔드 및 API의 사용량과 발생 비용을 한눈에 파악하여 예산 관리를 쉽게 하기 위함.
* **최종 목표:** '나의 서비스'와 '내가 사용하는 서비스'의 사용량, 백엔드 및 API 연결 상태를 한곳에 표시하는 대시보드 구축.

## 2. 핵심 기능 및 기술 스택 (Core & Stack)
* **주요 기술 스택 (AI Tool & Framework):**
  * IDE / AI: 안티그래비티 (Antigravity)
  * 연동 인프라 및 서비스: Supabase, Cloudflare R2, GitHub, Google Cloud (Storage, API)

* **핵심 기능 및 데이터 연동 로직 (MVP):**
  * **기능 1:** 서비스별 대시보드 시각화
  * **기능 2:** PAT(Personal Access Token) 기반 데이터 자동 연동
    * *[구현 로직 구상]*
      1. **Settings:** 계정 정보(PAT) 1회 입력 (예: Supabase Dashboard → Account → Access Tokens)
      2. **자동 실행 (프로젝트 목록 조회):** `API: GET https://api.supabase.com/v1/projects` → ["Healing Crew", "GhostShot", ...] 반환
      3. **자동 실행 (프로젝트별 사용량 조회):** `GET https://api.supabase.com/v1/projects/{ref}/usage` → DB 사용량, Storage 사용량, Edge Function 호출 수 등 연동
