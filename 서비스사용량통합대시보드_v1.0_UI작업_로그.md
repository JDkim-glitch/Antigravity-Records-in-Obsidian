# 🎯 V1.0 작업 일지: 대시보드 개선 및 관리자/데이터 연동 기능

* **적용된 개발 룰:** [[Rule_Dev]]
* **이번 버전의 핵심 목표:** 3단 레이아웃을 사용성 높은 2단으로 개편하고, 모의(Mock) 데이터가 아닌 API를 통한 실제 데이터 연동 기반 마련하기.

---

## 🛠️ 단계별 진행 과정 및 트러블슈팅 (Phases 1~12)

### Phase 1: 사용성 및 레이아웃 재구조화
* **작업 내용:** * 3단 레이아웃을 비대칭 2단 레이아웃(메인 콘텐츠 + 사이드바)으로 리팩토링.
  * 내비게이션 헤더 추가 ("Dashboard" 및 "Admin" 링크).
  * 메인 열에 "내 서비스" 및 "사용 중인 서비스 / AI" 그리드 형태로 표시, 사이드바 열에 캘린더, 할 일, GitHub 활동 표시.
* 🤖 **AI 프롬프트 지시:** *(예: "기존 3단 그리드를 좌측 메인과 우측 사이드바로 나누는 비대칭 2단 레이아웃으로 변경해 줘.")*
* 🛑 **트러블슈팅:** (안티그래비티가 레이아웃을 깰 때 어떻게 프롬프트를 수정했는지 기록)

### Phase 2: 할 일 목록 및 캘린더 연동
* **작업 내용:** * 캘린더 날짜 클릭 및 선택 상태 관리 구현.
  * 캘린더 하단에 할 일 목록 컴포넌트 생성 및 추가/완료/삭제 UI 구현.
  * 날짜별 작업 저장을 위한 로컬 스토리지 연동.

### Phase 3: 관리자 페이지 및 상태 관리
* **작업 내용:** * 라우터 설정 (`/`, `/admin`).
  * 영속성을 위해 데이터를 로컬 스토리지로 마이그레이션.
  * 관리자 페이지 폼 구현 ("내 서비스" 및 "AI 에이전트" 추가/수정/삭제).

### Phase 4 & 5: 서비스 세부 정보 확장 및 검증
* **작업 내용:** * 데이터 모델 업데이트 (프론트엔드, 백엔드, 깃허브 저장소 필드 추가) 및 관리자 페이지 입력 폼 반영.
  * 대시보드 카드 클릭 이벤트 및 세부 정보 모달 오버레이 구현.
  * 초기 설정 데이터에 예제 서비스 추가.
  * 뷰어, 할 일 전환, 메인 대시보드 반영 등 전체 시각적 검증 완료.

### Phase 6: 인라인 서비스 세부 정보 및 역방향 종속성
* **작업 내용:** * 모달 오버레이 방식 제거 -> 인라인 아이콘/배지를 사용해 상세 정보 표시로 UI 변경.
  * 선택 및 모달 관련 코드 제거 후 전체 서비스 목록 전달 구조로 변경.
  * Supabase 및 Cloudflare R2에 종속된 서비스를 동적으로 계산하여 표시하도록 구현.
* 🤖 **AI 프롬프트 지시:** "모달로 띄우는 방식은 불편해. 카드 내부에 인라인 아이콘으로 프론트엔드와 백엔드 스택을 바로 표시하도록 코드를 수정해 줘."

### Phase 7 & 8: 대시보드 상/하단 재구조화 및 M3 UI 적용
* **작업 내용:** * 상태(사용량 데이터, 로딩 등)를 상위(`App.tsx`)로 끌어올림. 새로고침 버튼 상단 바 이동.
  * 상단(서비스/캘린더) 및 하단(인프라/API/할 일)으로 컴포넌트 및 CSS 재구조화.
  * Material 3 Expressive 가이드라인 적용 (카드 테두리 반경 28px, 부드러운 그림자, 두꺼운 알약 형태 프로그레스 바).
  * Supabase 데이터베이스 사용량 프로그레스 바 구현.

### Phase 9 & 10: 내 서비스 마스터-디테일 재디자인 및 데이터 바인딩
* **작업 내용:** * 사용자 서비스를 위한 마스터-디테일 레이아웃 모듈 생성 및 그리드(250px 1fr) 디자인.
  * 할 일 열과 캘린더가 완벽히 정렬되도록 하단 그리드 비율 조정 (1fr 1fr 340px).
  * 객체 필드 확장 (수익, 가입자, 활성 유저, DB 메트릭 등).
* 🛑 **트러블슈팅 (선택 버그 해결):** * **이슈:** 마스터-디테일 뷰에서 항목 선택 시 제대로 데이터가 바인딩되지 않는 버그 발생.
  * **해결:** ID 기반 선택에서 서비스 이름(`service.name`) 기반 선택으로 바인딩을 변경하여 버그 수정 및 실제 데이터 렌더링 정상화.

### Phase 11: 메인/사이드바 레이아웃 병합 및 클라우드 연동 팝업
* **작업 내용:** * 듀얼 컬럼 레이아웃 리팩토링 및 사이드바 내부로 할 일 목록 이동.
  * 클라우드(Supabase, R2, Google Cloud) 연동을 위한 통합 모달(`IntegrationModal.tsx`) 생성.
  * API 키(PAT) 존재 여부에 따라 "연결하기" 버튼 또는 실제 사용량 렌더링 분기 처리.
* 🤖 **AI 프롬프트 지시:** "Supabase API 키를 입력받아 저장할 건데, 보안에 유의해서 처리하고 연동되지 않았을 때만 '연결하기' 모달을 띄워 줘."

### Phase 12: V1 최종 마무리 및 실데이터 렌더링
* **작업 내용:** * (사용자/본인 판단) V1 단계에 불필요한 모의 위젯 및 데이터 제거.
  * Edge Function에서 자동으로 가져온 실제 백엔드 사용량 데이터를 UI에 맵핑 및 연결 완료.
  * Git 저장소 초기화, V1 커밋 및 GitHub 푸시 완료.
