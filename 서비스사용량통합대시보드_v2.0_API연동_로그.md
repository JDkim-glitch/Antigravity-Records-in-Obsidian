# 🎯 V2.0 작업 일지: 서비스 자동 연동 및 모던 UI 개편 완료

* **적용된 개발 룰:** [[Rule_Dev]]
* **이번 버전의 핵심 목표:** 1차(V1)에서 구상했던 실제 데이터 연동(Supabase, Cloudflare R2) 자동화 구축, CORS 문제 해결, 그리고 가독성을 위한 UI/UX 전면 개편.

---

## 🛠️ 단계별 진행 과정 및 트러블슈팅 (Phases)

### Phase 1: 서비스 자동 탐색 시스템 구축 및 CORS 대응
* **작업 내용:** * **Supabase 연동:** 개인 액세스 토큰(PAT) 하나만 입력하면 모든 하위 프로젝트를 자동 탐색하여 대시보드에 개별 카드로 표시되도록 구현.
  * **Cloudflare R2 연동:** Account ID와 API 토큰을 사용하여 모든 버킷 목록을 가져오고 사용량을 실시간으로 집계. (참고: [[Cloudflare_R2_API_Guide]])
* 🛑 **트러블슈팅 (CORS 보안 정책 이슈):** * **이슈:** 브라우저(클라이언트)에서 외부 API를 직접 호출하려다 보니 CORS(Cross-Origin Resource Sharing) 차단 에러가 발생함.
  * **해결 (아키텍처 변경):** Supabase Edge Function을 활용하여 전용 프록시(`list-projects`) 서버를 배포함. 브라우저가 직접 API를 찌르지 않고 프록시를 통해 모든 외부 API 호출을 안전하게 중계하도록 설계하여 보안 정책 문제를 우회함. (참고: [[Supabase_Edge_Functions]], [[React_CORS_Proxy_Pattern]])
* 🤖 **AI 프롬프트 지시:** *(예: "브라우저에서 직접 API를 호출하니 CORS 에러가 나. Supabase Edge Function을 만들어서 이 호출을 중계하는 프록시 역할을 하도록 코드를 짜 줘.")*

### Phase 2: UI/UX 디자인 고도화 (디자인 피벗 반영)
* **작업 내용:** * 1차 작업의 픽셀 아트 디자인이 가독성이 떨어진다고 판단, 과감히 폐기하고 화이트 톤의 모던한 프리미엄 디자인으로 전면 개편.
  * 글래스모피즘(Glassmorphism)과 부드러운 애니메이션 효과를 적용하여 사용자 경험(UX) 향상.
  * 설정(Settings) 페이지를 탭 방식으로 구조화하여 관리 편의성 극대화.
* 🤖 **AI 프롬프트 지시:** *(예: "기존 픽셀 아트 스타일 CSS를 전부 걷어내 줘. 대신 화이트 톤 배경에 글래스모피즘(Glassmorphism) 효과를 넣고, 모던하고 깔끔한 UI로 전면 수정해 줘.")*

### Phase 3: 안정성 확보 및 예외 처리 (방어적 코딩)
* **작업 내용:** * 멀티 프로젝트 환경에서의 데이터 동기화 최적화를 위해 `useLiveUsage` 훅(Hook) 개선.
* 🛑 **트러블슈팅 (런타임 에러):** * **이슈:** 외부 API의 응답 구조가 변경되거나 값이 없을 때 `buckets.map is not a function` 등의 런타임 에러 발생하여 대시보드가 뻗는 현상 발견.
  * **해결:** 데이터가 배열이 아니거나 `null` / `undefined`일 경우를 대비하여 옵셔널 체이닝(`?.`) 및 기본값 처리를 하는 방어적 코딩(Defensive Coding)을 추가로 지시하여 안정성 확보.
